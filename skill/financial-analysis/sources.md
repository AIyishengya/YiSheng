# 信息源配置

本文档定义金融分析使用的可靠信息源及其时效性标准。

---

## 一、时效性分级标准

### 信息时效性等级

| 等级 | 有效时间 | 适用信息类型 | 更新策略 |
|------|----------|--------------|----------|
| **L1 实时** | 0-15分钟 | 股票行情、外汇、商品价格 | 必须实时更新，标注精确时间 |
| **L2 日级** | 0-24小时 | 交易数据、新闻快讯 | 每日更新，标注日期 |
| **L3 周级** | 0-7天 | 周度数据、短期分析 | 每周更新，注意过期风险 |
| **L4 月级** | 0-30天 | 月度经济数据、行业动态 | 每月更新，历史数据作参考 |
| **L5 季级** | 0-90天 | 财务报表、季报分析 | 季度更新，关注同比环比 |
| **L6 年级** | 0-365天 | 年报、行业报告、战略规划 | 年度更新，长期趋势分析 |
| **L7 永久** | 不限 | 公司公告、政策法规、历史事件 | 永久有效，注意最新修订 |

### 时效性衰减模型

```yaml
衰减公式:
  时效性系数 = exp(-λ × 超期天数)

衰减率 λ 设定:
  L1 实时数据: λ = 100  (超期1天几乎无效)
  L2 日级数据: λ = 10   (超期7天权重降至0.1%)
  L3 周级数据: λ = 3    (超期30天权重降至0.1%)
  L4 月级数据: λ = 1    (超期半年权重降至0.1%)
  L5 季级数据: λ = 0.3  (超期1年权重降至0.1%)
  L6 年级数据: λ = 0.1  (超期3年权重降至0.1%)
  L7 永久数据: λ = 0    (不衰减)

权重应用:
  信息有效权重 = 基础权重 × 时效性系数

  判断标准:
  - 系数 > 0.8: 信息完全有效 ✅
  - 系数 0.5-0.8: 信息部分有效 ⚠️
  - 系数 < 0.5: 信息可能过时 ❌ 建议更新
```

## 二、实时价格获取——完整回退链

> ⛔ **零容忍规则**：永远不允许使用估算或记忆中的价格。必须逐级尝试以下回退链直到成功，或在所有步骤失败后明确声明"价格获取失败"。

### A股价格获取回退链

```yaml
优先级 1 — WebSearch（同时搜索 2 个关键词）:
  查询1: "{股票名} {6位代码} 今日收盘价 {YYYY年MM月}"
  查询2: "SH{代码} 或 SZ{代码} 实时股价 {YYYY-MM-DD}"
  成功条件: 搜索结果中包含具体数字价格且来源可辨认

优先级 2 — WebFetch 新浪实时行情:
  上证: https://hq.sinajs.cn/list=sh{6位代码}
  深证: https://hq.sinajs.cn/list=sz{6位代码}
  返回格式: CSV字符串，第4字段为当前价，第3字段为昨收
  示例: sh600519 → 贵州茅台

优先级 3 — WebFetch 东方财富:
  URL: https://push2.eastmoney.com/api/qt/stock/get?secid={市场代码}.{股票代码}&fields=f43,f57,f58
  市场代码: 1=上证, 0=深证

优先级 4 — WebFetch 腾讯财经:
  URL: https://qt.gtimg.cn/q=sh{代码}  (上证)
  URL: https://qt.gtimg.cn/q=sz{代码}  (深证)

优先级 5 — WebSearch 最新财经新闻提取:
  查询: "{股票名} 股价 {YYYY-MM-DD}"
  注意: 仅接受新闻发布时间在 2 个交易日内的数据，必须标注来源和时间

失败处理:
  frontmatter: current_price: null
  报告内写明: "⚠️ 当前股价：获取失败（已尝试5个信源，查询时间 {datetime}）"
```

### 美股/ETF 价格获取回退链

```yaml
优先级 1 — WebSearch:
  查询1: "{Ticker} stock price {YYYY-MM-DD}"
  查询2: "{公司名} {Ticker} share price today"
  成功条件: 搜索结果中有具体价格数字

优先级 2 — WebFetch Yahoo Finance:
  URL: https://finance.yahoo.com/quote/{Ticker}/
  提取: "regularMarketPrice" 或页面中的当前价格

优先级 3 — WebFetch Google Finance:
  URL: https://www.google.com/finance/quote/{Ticker}:NYSE
  URL: https://www.google.com/finance/quote/{Ticker}:NASDAQ

优先级 4 — WebFetch Stock Analysis:
  URL: https://stockanalysis.com/stocks/{小写ticker}/

优先级 5 — WebSearch 最新新闻提取:
  查询: "{Ticker} stock {YYYY Month} price"
  同上，仅接受 2 交易日内的数据

失败处理: 同上，声明获取失败
```

### 价格标注格式（强制）

所有报告中的价格必须使用以下格式标注来源和时间：

```markdown
当前股价：¥1,650.00（来源：新浪财经，截至 2026-04-12 15:00 A股收盘）
Current Price: $88.22（来源：Yahoo Finance，截至 2026-04-12 16:00 EST）
当前股价：HK$92.30（来源：Yahoo Finance，截至 2026-04-11 收盘，节假日数据）
⚠️ 当前股价：获取失败（已尝试5个信源，查询时间 2026-04-12 10:30，相关估值数据暂缺）
```

## 六、信息源使用原则

### 1. 多源交叉验证

```yaml
验证原则:
  - 关键数据至少使用2个独立信源验证
  - 官方信源优先（如巨潮资讯、SEC）
  - 冲突时以权威性更高的信源为准
  - 记录所有使用的信源链接
```

### 2. 时效性优先

```yaml
时效原则:
  - 优先使用最新发布的数据
  - 过期数据必须标注时间并警告
  - 实时数据必须在报告中标注获取时间
  - 定期检查数据源更新频率
```

### 3. 可信度评估

```yaml
可信度标准:
  ⭐⭐⭐⭐⭐ 官方来源、权威机构
  ⭐⭐⭐⭐   专业媒体、知名数据商
  ⭐⭐⭐     一般媒体、自媒体（需验证）
  ⭐⭐       存疑来源、需交叉验证
  ⭐         不可靠来源、不建议采用
```
