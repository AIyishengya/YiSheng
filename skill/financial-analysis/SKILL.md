# 金融分析 Skill v2.0

专业的金融分析与研究技能，提供从信息收集到研究报告生成的完整流程。

## 核心能力

| 能力 | 描述 |
|------|------|
| **智能识别** | 自动识别股票名、公司名、时政事件、行业板块、基金/ETF、债券等输入类型 |
| **并行信息收集** | 多路并发搜索，从可靠信源同时获取财务、新闻、公告、研报 |
| **权威框架分析** | 应用巴菲特、格雷厄姆、达利欧等权威分析框架 |
| **系统化技术分析** | RSI、MACD、布林带、均线系统等完整技术指标体系 |
| **量化风险评分** | 7维度加权评分（1-10），替代纯定性描述 |
| **历史对比追踪** | 自动读取同标的历史报告，生成变化追踪与预测回顾 |
| **预测回溯验证** | 对历史预测进行回测，量化分析准确率 |
| **组合对比分析** | 支持多标的横向对比，竞争力雷达图 |
| **观察清单管理** | 维护关注标的、关键价位、事件日历 |
| **宏观周报** | 全球宏观环境周报，个股报告可引用 |
| **迭代深化研究** | 多轮查询与思考，逐步完善分析 |
| **时效性管理** | 智能判断信息有效期，确保报告准确性 |
| **Obsidian 深度集成** | 双链引用、结构化 frontmatter、Dataview 支持 |

## 分析流程

### Phase 0 — 日期确认（强制）

在执行任何搜索前，**必须**确认当前日期：
1. 从系统上下文中读取 `currentDate`
2. 确定当前年份，用于所有搜索查询
3. 所有搜索词中用实际年份替换 `{当前年份}`

```yaml
正确搜索格式示例:
  财报: "贵州茅台 2026年年报"
  评级: "腾讯 分析师评级 2026年4月"
  新闻: "美联储利率决议 2026年4月"
```

详见：[时效性管理](./时效性管理.md)

### Phase 0.5 — 历史报告检索（自动）

**在 Phase 1 之前**，自动检查是否有同标的历史分析报告：

```yaml
历史检索:
  1. Glob 搜索: /财务分析/{标的名称}/*.md
  2. 若有历史报告:
     - 读取最近一篇的完整内容
     - 提取关键指标（股价、评级、目标价、风险评分）
     - 标记需要在最终报告中生成「变化追踪」章节
  3. 若无历史报告:
     - 跳过，正常流程
```

详见：[历史对比分析](./comparison-history.md)

### Phase 0.8 — 真实价格获取（强制，零容忍估算）

> ⛔ **绝对禁止**：在报告任何位置使用估算、推测、训练数据记忆中的股价。
> 即使搜索暂时失败，也**绝不**使用近似值，必须继续尝试直到获取真实价格或明确声明"无法获取"。

#### 获取流程（按优先级逐级尝试，直到成功）

```yaml
Step 1 - WebSearch（主力）:
  同时搜索至少 2 个独立信源:
  A股:
    - 搜索: "{股票名} {股票代码} 今日股价 {当前年份}{当前月份}"
    - 搜索: "{股票代码} 实时行情 收盘价"
  美股:
    - 搜索: "{Ticker} stock price today {当前年份}"
    - 搜索: "{Ticker} current price nasdaq/nyse"
  港股:
    - 搜索: "{股票代码}.HK 今日股价 {当前年份}"
    - 搜索: "{股票名} 港股 实时行情"

Step 2 - WebFetch（备用，当搜索结果不含具体价格时）:
  A股备用信源（按序尝试）:
    1. https://finance.sina.com.cn/realstock/company/{代码}/nc.shtml
    2. https://quote.eastmoney.com/{代码}.html
    3. https://xueqiu.com/S/{代码}
  美股备用信源（按序尝试）:
    1. https://finance.yahoo.com/quote/{Ticker}/
    2. https://www.google.com/finance/quote/{Ticker}:NASDAQ
    3. https://stockanalysis.com/stocks/{ticker}/
  港股备用信源（按序尝试）:
    1. https://finance.yahoo.com/quote/{代码}.HK/
    2. https://www.aastocks.com/tc/stocks/quote/quick-quote.aspx?symbol={代码}

Step 3 - 搜索新闻中的最新价格（当行情页无法访问时）:
  - 搜索: "{股票名} 股价 {当前年份}{当前月份}{当前日}"
  - 搜索最近24小时内的财经新闻，提取文中提到的最新价格
  - 注意：必须确认新闻中价格的发布时间，不接受超过 2 个交易日的数据

Step 4 - 放弃估算，如实声明:
  若以上3步全部失败（无法访问任何信源）:
  - 在报告"基本信息"章节明确写明:
    "⚠️ 股价：获取失败（搜索于 {当前日期}，已尝试 {N} 个信源）"
  - frontmatter 中 current_price 填写 null
  - 所有依赖当前股价的估值计算（PE、目标价上涨空间等）标注"待价格数据更新后计算"
  - 绝不填写任何数字作为当前股价的近似值
```

#### 价格数据质量验证

获取到价格后，必须验证其有效性：

```yaml
验证规则:
  1. 时间验证: 价格数据发布时间不超过 2 个交易日
  2. 来源标注: 必须标注"来源: {信源名称}，截至 {YYYY-MM-DD HH:MM}"
  3. 合理性检验: 若多个信源价格差距 > 5%，需说明差异原因（如盘中波动、时区差）
  4. 交易状态说明:
     - 盘中数据: 标注"盘中实时"
     - 收盘数据: 标注"收盘价 {日期}"
     - 节假日/停牌: 标注"停牌"或"最近收盘价 {日期}"

禁止行为:
  ❌ 使用训练数据中记忆的历史价格
  ❌ 使用"约"、"大约"、"估计"、"参考"等模糊表述作为股价
  ❌ 从历史报告中复用上次的股价而不重新查询
  ❌ 因"搜索一次未成功"就放弃获取（必须至少尝试 3 个信源）
  ❌ 用 PE × EPS 反推当前股价替代真实行情
```

### Phase 1 — 并行信息收集

**识别输入类型后，立即发起并行搜索：**

```yaml
股票分析（5路并行）:
  - 路径A: 实时股价 + 基本行情
  - 路径B: 最新财务报表 + 关键指标
  - 路径C: 近期券商研报 + 机构评级
  - 路径D: 近期重要公告 + 新闻动态
  - 路径E: 技术面数据（均线、RSI、MACD）

公司研究（3路并行）:
  - 路径A: 公司基本信息 + 主营业务
  - 路径B: 竞争格局 + 行业地位
  - 路径C: 财务数据 + 估值指标

行业分析（3路并行）:
  - 路径A: 行业规模 + 增长率 + 竞争格局
  - 路径B: 政策监管 + 行业趋势
  - 路径C: 龙头公司数据 + 估值对比

基金分析（3路并行）:
  - 路径A: 基金净值 + 业绩表现 + 风险指标
  - 路径B: 持仓明细 + 行业分布
  - 路径C: 基金经理 + 同类排名

债券分析（3路并行）:
  - 路径A: 债券基本信息 + 收益率
  - 路径B: 发行人信用分析 + 财务状况
  - 路径C: 利率环境 + 信用利差走势

对比分析: 参见 comparison.md
宏观周报: 参见 macro-weekly.md（6路并行）
```

搜索词按市场类型选择对应模板，详见：[信息源配置 - 搜索词模板](./sources.md)

### Phase 2 — 时效性校验

检查所有获取信息的发布时间，标注时效状态：
- ✅ 有效（系数 ≥ 0.8）
- ⚠️ 较旧（0.5-0.8），标注警告
- ❌ 过期（< 0.5），触发补充搜索

### Phase 3 — 框架分析

根据分析类型自动选择框架：

| 场景 | 主要框架 | 辅助框架 |
|------|----------|----------|
| 长期价值投资 | 巴菲特护城河 + DCF | 格雷厄姆安全边际 |
| 成长股分析 | PEG + PS估值 | 波特五力 |
| 宏观/时政 | 达利欧周期 + 美林时钟 | PEST |
| 行业比较 | 波特五力 + 相对估值 | SWOT |
| 短线/技术面 | RSI + MACD + 布林带 | 成交量 + ATR |
| 对比分析 | 相对估值 + 波特五力 | PEG + SWOT |

详见：[分析框架详解](./frameworks.md)

### Phase 3.5 — 量化风险评分

对标的进行 7 维度量化评分（1-10分）：

```yaml
评分维度:
  - 财务风险 (25%)
  - 估值风险 (20%)
  - 市场风险 (15%)
  - 经营风险 (15%)
  - 政策/监管风险 (10%)
  - 流动性风险 (5%)
  - 特殊事件风险 (10%)
```

详见：[框架详解 - 量化风险评分](./frameworks.md)

### Phase 4 — 迭代深化

识别信息缺口，补充关键数据，修正初步结论。

### Phase 5 — 报告生成

输出结构化报告，附时效性声明。

**宏观引用**：检查是否存在本周宏观周报，若存在则用双链引用代替重复搜索。

**观察清单更新**：报告生成后，自动检查并更新观察清单。

## Obsidian 集成规范

### Frontmatter 结构化字段

所有报告的 frontmatter 必须包含以下字段，支持 Obsidian Dataview 查询：

```yaml
---
title: 报告标题
date: YYYY-MM-DD
type: 财务分析/行业分析/对比分析/基金分析/债券分析/宏观周报/回测验证
tags:
  - 标签1
  - 标签2
# Dataview 查询字段
ticker: CRCL          # 股票/基金/债券代码
market: NYSE/SSE/SZSE/HKEX  # 交易所
rating: 买入/增持/持有/减持/卖出/观望  # 投资评级
target_price: 127     # 目标价（数字）
current_price: 88.22  # 分析时股价（数字）
risk_score: 6         # 风险评分 1-10（数字）
sector: 金融科技       # 行业板块
---
```

### Wiki Link 双链规则

在报告中引用其他分析报告时，使用 Obsidian wiki link：

```markdown
# 引用历史报告
> 上次分析：[[Circle-2026-04-12]]

# 引用宏观周报
> 详见 [[宏观周报-2026W15]]

# 在对比分析中引用个股报告
详细分析：[[阿里巴巴-2026-04-11]]

# 在观察清单中引用
最近分析：[[Circle-2026-04-12]]
```

### Dataview 查询示例

用户可在 Obsidian 中使用以下 Dataview 查询：

```dataview
TABLE ticker, rating, risk_score, current_price, target_price
FROM "财务分析"
WHERE type = "财务分析"
SORT date DESC
```

## 信息时效性标准（快速参考）

| 信息类型 | 等级 | 有效期 | 关键规则 |
|----------|------|--------|----------|
| 实时行情 | L1 | 15分钟 | 必须标注精确获取时间 |
| 收盘数据/新闻 | L2 | 24小时 | 标注数据日期 |
| 月度经济数据 | L4 | 30天 | 关注发布日期 |
| 财务报表/机构评级 | L5 | 90天 | **评级必须标注发布日期** |
| 年度报告/行业报告 | L6 | 365天 | 标注报告年度 |
| 公司公告/政策 | L7 | 永久 | 关注最新修订 |

