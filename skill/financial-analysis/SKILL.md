# 金融分析 Skill v2.1

## 目的

此 skill 编排端到端的金融分析流程，从标的识别、信息收集、框架分析到报告生成。

### 核心能力

| 能力 | 描述 |
|------|------|
| **智能识别** | 自动识别股票、公司、时政事件、行业板块、基金/ETF、债券等输入类型 |
| **权威框架分析** | 应用巴菲特、格雷厄姆、达利欧、波特五力等权威分析框架 |
| **系统化技术分析** | RSI（相对强弱指标）、MACD（指数平滑异同移动平均线）、布林带、均线系统等完整技术指标体系 |
| **量化风险评分** | 7 维度加权评分（1-10），替代纯定性描述 |
| **历史对比追踪** | 自动检索同标的历史报告，生成变化追踪与预测回顾 |
| **预测回溯验证** | 对历史预测进行回测，量化分析准确率 |
| **组合对比分析** | 多标的横向对比，竞争力雷达图 |
| **观察清单管理** | 维护关注标的、关键价位、事件日历 |
| **宏观周报** | 全球宏观环境周报，个股报告可引用 |
| **时效性管理** | 智能判断信息有效期，指数衰减模型 |
| **Obsidian 集成** | 双链引用、结构化 frontmatter、Dataview 支持 |

## 何时激活

当出现以下场景时激活此 skill：

- 用户请求分析特定股票、公司、行业板块、基金/ETF、债券
- 用户请求分析时政事件对市场、行业或个股的影响
- 用户请求对比多个标的（如"对比 A vs B"、"比较这几只股票"）
- 用户请求回测历史预测准确率
- 用户请求生成宏观周报
- 用户请求管理观察清单（添加、移除、更新）
- 用户请求进行估值分析、财务分析、技术面分析
- 用户询问投资建议、风险评估、行业趋势

## 关键概念

### 前置条件

- 能够访问互联网搜索金融数据（行情、财报、新闻）
- 能够读写文件系统以保存/检索分析报告
- 当前日期已确认（用于所有搜索查询的年份标注）

### 完成定义

- 报告包含所有必需章节（摘要、财务分析、估值、风险评分、投资建议、时效声明）
- 股价数据来源已标注（非估算）
- 所有数据标注时效状态
- 报告已保存至文件系统

### 分析模式

| 模式 | 触发词 | 适用场景 |
|------|--------|----------|
| **完整分析** | 默认 | 投资决策、深度研究 |
| **快速分析** | "快速"、"简要"、"概览" | 初步了解、市场动态 |
| **对比分析** | "对比"、"比较"、"vs"、"PK" | 同行业标的选择 |
| **回测验证** | "回测"、"验证" | 检验历史预测准确率 |
| **宏观周报** | "宏观周报"、"本周宏观" | 宏观环境总览 |

### 条件能力模式

#### 如果你有网络搜索能力

- 通过搜索获取实时行情、财报数据、机构评级
- 使用至少 2 个独立信源交叉验证关键数据
- 优先使用官方信源（巨潮资讯、SEC、港交所）

#### 如果你有网页抓取能力

- 当搜索结果不含具体价格时，尝试访问金融数据网站
- A 股备用信源：新浪财经、东方财富、腾讯财经
- 美股备用信源：Yahoo Finance、Google Finance、Stock Analysis

#### 如果你有文件系统读写能力

- 保存分析报告至结构化目录
- 检索同标的历史报告生成变化追踪
- 维护观察清单文件

#### 如果你仅有只读能力

- 生成完整分析报告供用户审阅
- 报告中标注需要用户手动保存的位置
- 无法检索历史报告，跳过变化追踪章节

#### 如果无网络访问能力

- 明确告知用户无法获取实时数据
- 提示用户手动提供最新行情数据
- 所有价格字段标注"⚠️ 数据待更新"
- 仅基于用户提供的数据进行分析

## 应用步骤

### 阶段 1：日期确认与准备

在执行任何搜索前，确认当前日期（YYYY-MM-DD），所有搜索查询必须包含正确年份，避免获取过期数据。

**关键规则**：
- 所有搜索词中用实际年份替换模糊时间词（"今年" → "2026 年"）
- 机构评级必须标注发布日期（有效期 90 天）
- 财报数据标注报告期（如"2025 年年报"而非"最新财报"）

详见：[时效性管理](./references/time-validity.md)

### 阶段 2：历史报告检索

检查是否存在同标的的历史分析报告：
- 若有：读取最近一篇，提取关键指标（股价、评级、目标价、风险评分），在最终报告中生成「变化追踪」章节
- 若无：跳过，正常流程

详见：[历史对比分析](./references/comparison-history.md)

### 阶段 3：价格数据获取（零容忍估算）

> ⛔ **绝对禁止**：使用估算、推测、训练数据记忆中的股价。必须获取真实价格或明确声明"无法获取"。

**获取优先级**：
1. 通过搜索能力查询至少 2 个独立信源
2. 若搜索结果不含价格，尝试访问金融数据网站
3. 搜索最近 24 小时财经新闻提取价格
4. 若全部失败：在报告中明确声明"⚠️ 股价：获取失败"，所有估值计算标注"待价格数据更新后计算"

**价格标注格式**（强制）：
```text
当前股价：¥1,650.00（来源：新浪财经，截至 2026-04-12 15:00 A股收盘）
⚠️ 当前股价：获取失败（已尝试 3 个信源，查询时间 2026-04-12 10:30）
```

**禁止行为**：
- ❌ 使用训练数据中记忆的历史价格
- ❌ 使用"约"、"大约"、"估计"等模糊表述作为股价
- ❌ 从历史报告中复用上次的股价而不重新查询
- ❌ 用 PE × EPS 反推当前股价替代真实行情

详见：[信息源配置 - 价格获取回退链](./references/sources.md)

### 阶段 4：并行信息收集

识别输入类型后，发起多维度信息收集：

**股票分析**（5 维度）：
- 实时股价 + 基本行情
- 最新财务报表 + 关键指标
- 近期券商研报 + 机构评级
- 近期重要公告 + 新闻动态
- 技术面数据（均线、RSI、MACD）

**行业分析**（3 维度）：
- 行业规模 + 增长率 + 竞争格局
- 政策监管 + 行业趋势
- 龙头公司数据 + 估值对比

**基金/ETF 分析**（3 维度）：
- 基金净值 + 业绩表现 + 风险指标
- 持仓明细 + 行业分布
- 基金经理 + 同类排名

**债券分析**（3 维度）：
- 债券基本信息 + 收益率
- 发行人信用分析 + 财务状况
- 利率环境 + 信用利差走势

**对比分析**：详见 [组合对比分析](./references/comparison.md)

**宏观周报**：详见 [宏观周报](./references/macro-weekly.md)

搜索词模板详见：[信息源配置 - 搜索词模板](./references/sources.md)

### 阶段 5：时效性校验

检查所有获取信息的发布时间，标注时效状态：
- ✅ 有效（衰减系数 ≥ 0.8）
- ⚠️ 较旧（0.5-0.8），标注警告
- ❌ 过期（< 0.5），触发补充搜索

详见：[时效性管理 - 衰减模型](./references/time-validity.md)

### 阶段 6：框架分析

根据分析类型自动选择框架：

| 场景 | 主要框架 | 辅助框架 |
|------|----------|----------|
| 长期价值投资 | 巴菲特护城河 + DCF | 格雷厄姆安全边际 |
| 成长股分析 | PEG + PS 估值 | 波特五力 |
| 宏观/时政 | 达利欧周期 + 美林时钟 | PEST |
| 行业比较 | 波特五力 + 相对估值 | SWOT |
| 短线/技术面 | RSI + MACD + 布林带 | 成交量 + ATR |
| 对比分析 | 相对估值 + 波特五力 | PEG + SWOT |

详见：[分析框架详解](./references/frameworks.md)

### 阶段 7：量化风险评分

对标的进行 7 维度量化评分（1-10 分）：

```yaml
评分维度:
  - 财务风险 (25%)
  - 估值风险 (20%)
  - 市场风险 (15%)
  - 经营风险 (15%)
  - 政策/监管风险 (10%)
  - 流动性风险 (5%)
  - 特殊事件风险 (10%)
```

评分解读：
- 1-2 分：极低风险 🟢（适合所有类型投资者，仓位上限 15%）
- 3-4 分：低风险 🟢（仓位上限 10%）
- 5-6 分：中等风险 🟡（稳健型以上，仓位上限 5-8%）
- 7-8 分：高风险 🟠（激进型，仓位上限 3-5%）
- 9-10 分：极高风险 🔴（仅投机，仓位上限 ≤2%）

详见：[框架详解 - 量化风险评分](./references/frameworks.md)

### 阶段 8：迭代深化与报告生成

识别信息缺口，补充关键数据，修正初步结论。

输出结构化报告，附时效性声明。

**宏观引用**：检查是否存在本周宏观周报，若存在则用双链引用代替重复搜索。

**观察清单更新**：报告生成后，自动检查并更新观察清单。

报告模板详见：[报告模板](./references/templates.md)

## 示例

### 股票分析完整流程

**输入**："分析贵州茅台的投资价值"

**流程**：
1. 确认当前日期（2026-06-15）
2. 检索历史报告（假设无历史记录）
3. 获取实时股价：¥1,680.00（来源：东方财富，截至 2026-06-15 15:00）
4. 并行收集：财务报表、机构评级、新闻动态、技术面数据
5. 时效性校验：所有数据 ✅ 有效
6. 框架分析：巴菲特护城河 + DCF + 波特五力
7. 风险评分：综合 4/10 🟢
8. 生成报告并保存至 `/财务分析/贵州茅台/贵州茅台-2026-06-15.md`

**输出**：完整分析报告，包含摘要、财务分析、估值分析、技术面分析、行业分析、风险评分、投资建议、时效声明。

### 对比分析流程

**输入**："对比 贵州茅台 vs 五粮液"

**流程**：
1. 识别两个标的，确定行业（白酒）
2. 对每个标的并行收集数据
3. 多维对比：基本面、估值、市场表现、竞争力、风险
4. 生成对比报告，包含竞争力雷达图和综合排名

## 检查清单

完成分析后，逐项验证以下内容：

### 数据质量

- [ ] 当前股价已通过实际查询获取（非估算），并标注来源和时间
- [ ] 所有搜索查询包含正确年份（避免使用"今年"、"去年"等模糊时间词）
- [ ] 机构评级标注发布日期，超过 90 天加注警告
- [ ] 关键数据至少使用 2 个独立信源交叉验证

### 报告完整性

- [ ] 包含所有必需章节（摘要、财务分析、估值分析、技术面分析、行业分析、风险评分、投资建议、时效声明）
- [ ] 如有历史报告，生成「变化追踪」章节
- [ ] 风险评分包含 7 维度明细（财务、估值、市场、经营、政策、流动性、特殊事件）
- [ ] 投资建议区分不同投资者类型（价值型、成长型、稳健型等）

### 时效性管理

- [ ] 所有数据标注时效状态（✅有效 / ⚠️较旧 / ❌过期）
- [ ] 过期数据（衰减系数 <0.5）已触发补充搜索或在报告中明确标注
- [ ] 报告包含完整的数据时效性声明表（数据项、来源、发布时间、时效状态、更新建议）

### 格式规范

- [ ] 报告 frontmatter 包含所有必需字段（title、date、type、tags、ticker、market、rating、target_price、current_price、risk_score、sector）
- [ ] 引用历史报告时使用 wiki link 格式（如 `[[Circle-2026-04-12]]`）
- [ ] 价格数据标注获取时间和来源（如"来源：新浪财经，截至 2026-04-12 15:00"）

## 常见陷阱

### 陷阱 1：使用估算股价

**症状**：报告中使用"约¥100"、"估计$50"等模糊表述

**原因**：搜索暂时失败时，试图用训练数据中的历史价格填充

**修复**：绝对禁止估算。必须获取真实价格或明确声明"⚠️ 股价：获取失败"，所有估值计算标注"待价格数据更新后计算"

### 陷阱 2：过期机构评级

**症状**：引用 1 年前的机构评级作为当前投资建议

**原因**：未标注评级发布日期，未检查时效性

**修复**：所有评级必须标注发布日期。超过 90 天加注"⚠️ 较旧"警告，超过 180 天视为过期不建议引用

### 陷阱 3：年份错误

**症状**：搜索"特斯拉 2025 年财报"但实际已是 2026 年

**原因**：未确认当前日期，使用模糊时间词（"今年"、"去年"）

**修复**：所有搜索查询必须包含明确年份（如"2026 年"），避免使用相对时间词

### 陷阱 4：技术面分析不完整

**症状**：仅列出 RSI 数值，未分析背离、超买超卖区间

**原因**：未遵循技术面分析模板的完整结构

**修复**：使用标准技术面分析模板，包含均线系统、动量指标、布林带、成交量、关键价位等完整章节

详见：[框架详解 - 技术分析报告模板](./references/frameworks.md)

### 陷阱 5：忽略时效性衰减

**症状**：使用 6 个月前的行业数据作为当前分析依据

**原因**：未应用时效性衰减模型，未标注数据有效期

**修复**：所有数据标注发布时间和时效等级（L1-L7），应用衰减系数判断有效性

详见：[时效性管理](./references/time-validity.md)
