# 预测回溯验证模块

## 概述

对历史分析报告中的预测和操作建议进行系统性回测，量化分析准确率，持续改进分析质量。

## 触发方式

```
/financial-analysis 回测 {标的名称}
/financial-analysis 回测 全部
/financial-analysis 验证 {标的名称}-{日期}
```

## 回测流程

### Step 1 — 收集历史预测

```yaml
扫描目标:
  路径: /财务分析/{标的名称}/*.md
  提取内容:
    - 分析日期
    - 当时股价
    - 价格区间预测（乐观/基准/悲观）
    - 操作建议（买入/持有/卖出）
    - 止损止盈位
    - 关键催化剂预判
```

### Step 2 — 获取实际结果

```yaml
对每条历史预测:
  - 搜索预测时间窗口内的实际股价走势
  - 确认催化剂事件是否发生及实际影响
  - 获取预测期后的实际收益率
```

### Step 3 — 量化评估

```yaml
评估维度:
  价格预测准确度:
    计算: |实际价格 - 预测中值| / 预测中值 × 100%
    评级:
      - A: 偏差 < 5%
      - B: 偏差 5-15%
      - C: 偏差 15-30%
      - D: 偏差 > 30%

  方向判断准确率:
    计算: 正确预测涨跌方向的次数 / 总预测次数
    评级:
      - A: > 70%
      - B: 50-70%
      - C: < 50%

  操作建议回报:
    计算: 按建议操作后的实际回报率
    对比: 同期大盘基准收益率（SPY / 沪深300）

  风险预警准确度:
    计算: 预警的风险事件中实际发生的比例
```
