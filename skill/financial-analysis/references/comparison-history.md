# 历史对比分析模块

## 概述

当对同一标的进行多次分析时，自动检索历史报告，生成「变化追踪」章节，帮助投资者理解标的在不同时间点的演变。

## 触发条件

在 Phase 1 信息收集**之前**，执行以下检查：

```yaml
历史报告检索:
  1. 搜索同标的的历史分析报告（路径：/财务分析/{标的名称}/*.md）
  2. 若找到历史报告:
     - 读取最近一篇报告的完整内容
     - 提取关键指标（股价、评级、目标价、风险评分）
     - 在最终报告中生成「变化追踪」章节
  3. 若无历史报告:
     - 跳过此步骤，按正常流程执行
```

## 变化追踪章节模板

在报告的「摘要」之后、「基本信息」之前插入：

```markdown
## 📊 变化追踪（对比上次分析）

> 上次分析日期：{上次日期} | 间隔天数：{天数}

| 指标 | 上次（{上次日期}） | 本次（{本次日期}） | 变化 | 判断 |
|------|-------------------|-------------------|------|------|
| 股价 | ${上次价格} | ${本次价格} | {涨跌幅}% | {趋势判断} |
| 分析师目标均价 | ${上次目标} | ${本次目标} | {变化幅度} | {共识变化} |
| 风险评分 | {上次评分}/10 | {本次评分}/10 | {变化} | {风险趋势} |
| 核心评级 | {上次评级} | {本次评级} | {变化} | — |
| USDC流通量/营收等关键指标 | {上次数据} | {本次数据} | {变化} | {趋势判断} |

### 预测回顾

| 上次预测 | 实际结果 | 准确度 |
|----------|----------|--------|
| {上次操作建议摘要} | {实际市场表现} | {评价} |
| {上次价格区间预测} | {实际价格走势} | {偏差百分比} |

### 观点变化说明

{简要说明本次分析与上次的主要观点差异及原因}
```

## 关键指标提取规则

从历史报告中提取以下字段（基于 frontmatter 和正文）：

```yaml
必提取字段:
  - ticker: 股票代码
  - price: 当时股价
  - rating: 综合评级（强烈买入/买入/持有/卖出）
  - target_price: 分析师目标均价
  - risk_score: 风险评分（1-10）
  - action: 操作建议摘要
  - price_prediction: 价格区间预测

可选字段（按标的类型）:
  - revenue: 最近一期营收
  - pe_ratio: 市盈率
  - market_cap: 市值
  - key_metric: 标的特有核心指标（如 USDC 流通量、DAU 等）
```

## 多次分析趋势

当历史报告 ≥ 3 篇时，额外生成趋势摘要：

```markdown
### 历史分析趋势

| 日期 | 股价 | 评级 | 风险 | 关键事件 |
|------|------|------|------|----------|
| {日期1} | ${价格1} | {评级1} | {风险1}/10 | {事件1} |
| {日期2} | ${价格2} | {评级2} | {风险2}/10 | {事件2} |
| {日期3} | ${价格3} | {评级3} | {风险3}/10 | {事件3} |

**趋势判断**：{基于多次分析的整体趋势判断}
```

## Obsidian 双链

在变化追踪章节中，引用历史报告使用 wiki link：

```markdown
> 上次分析：[[{标的名称}-{上次日期}]]
```
