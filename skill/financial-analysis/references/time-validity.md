# 信息时效性管理

本文档详细说明金融分析中的信息时效性管理机制，确保研究报告的准确性和可靠性。

---

## 〇、搜索前日期确认（强制流程）

### 核心原则

**所有互联网搜索必须先确认当前日期**，确保搜索查询使用正确的年份和时间范围。

### 日期确认流程

```yaml
步骤 1 - 获取当前日期:
  操作: 在执行任何网络搜索前，确认当前日期
  输出格式: YYYY-MM-DD（如：2026-03-14）
  来源优先级:
    1. 系统上下文中的 currentDate
    2. 用户明确指定的日期
    3. 询问用户确认日期

步骤 2 - 确认年份:
  操作: 明确当前年份用于搜索查询
  示例: 当前年份 = 2026
  注意: 年份错误会导致搜索到过期数据

步骤 3 - 校验特殊时间点:
  检查项:
    - 是否为财报披露期（1月/4月/7月/10月）
    - 是否为年报披露截止期（4月30日前）
    - 是否为重大政策发布窗口
    - 是否为交易日/节假日
```

### 搜索查询日期规范

```yaml
规则 1 - 年份强制包含:
  错误示例: "贵州茅台 最新财报"
  正确示例: "贵州茅台 2025年年报" 或 "贵州茅台 2026年最新消息"

规则 2 - 时效敏感信息的日期范围:
  财报类:
    - 查询格式: "[公司名] [年份]年[季度]财报"
    - 示例: "腾讯 2025年Q4财报"

  评级类:
    - 查询格式: "[公司名] 分析师评级 [年份]年[月份]"
    - 示例: "贵州茅台 分析师评级 2026年3月"

  新闻类:
    - 查询格式: "[主题] 最新 [年份]年[月份]"
    - 示例: "美联储加息 最新 2026年3月"

规则 3 - 动态数据的时间戳:
  股价数据:
    - 必须标注获取时间
    - 格式: "截至 YYYY-MM-DD HH:MM"
    - 示例: "截至 2026-03-14 15:00"

规则 4 - 避免歧义词汇:
  避免使用:
    - "今年" → 明确年份
    - "去年" → 明确年份
    - "最近" → 明确时间范围
    - "最新" → 结合具体时间
```

### 搜索日期校验检查表

```markdown
在执行搜索前，确认以下内容：

- [ ] 当前日期已确认：YYYY-MM-DD
- [ ] 当前年份已确认：YYYY
- [ ] 搜索查询包含正确年份
- [ ] 时效敏感信息包含月份范围
- [ ] 避免使用模糊时间词汇
```

### 常见日期错误案例

```yaml
案例 1 - 年份错误:
  错误: 搜索 "特斯拉 2025年财报"（实际已到 2026年）
  后果: 获取到过期一年的数据
  正确: 搜索 "特斯拉 2025年财报 发布时间 2026年"

案例 2 - 季度错误:
  错误: 搜索 "腾讯 Q1财报"（未指定年份）
  后果: 可能获取到多年前的 Q1 数据
  正确: 搜索 "腾讯 2026年 Q1财报"

案例 3 - 新闻时效性:
  错误: 搜索 "美联储利率决议 最新"
  后果: 可能获取到旧决议信息
  正确: 搜索 "美联储利率决议 2026年3月"
```

## 一、时效性分级体系

### 信息类型与有效区间

| 信息类型 | 等级 | 有效区间 | 更新频率 | 过期表现 |
|----------|------|----------|----------|----------|
| **实时行情** | L1 | 15分钟 | 秒级 | 数据失去参考价值 |
| **日内交易数据** | L2 | 24小时 | 日级 | 需重新获取当日收盘数据 |
| **周度数据** | L3 | 7天 | 周级 | 需确认是否有新周报 |
| **月度经济数据** | L4 | 30天 | 月级 | 关注最新发布日期 |
| **季度财务报表** | L5 | 90天 | 季级 | 对比最新报告期 |
| **机构评级** | L5 | 90天 | 不定期 | **必须标注评级日期** |
| **研报观点** | L5 | 180天 | 不定期 | 超过6个月需警告 |
| **年度报告** | L6 | 365天 | 年级 | 关注年报发布时间 |
| **行业报告** | L6 | 365天 | 年级 | 超过1年建议更新 |
| **公司公告** | L7 | 永久 | 不定期 | 注意是否有修订版本 |
| **政策法规** | L7 | 永久 | 不定期 | 注意最新修订日期 |

### 时效性等级详解

```yaml
L1 实时级 (有效期: 0-15分钟):
  适用:
    - 股票实时行情
    - 外汇实时报价
    - 商品期货价格
    - 加密货币价格
  处理策略:
    - 必须标注精确获取时间
    - 盘中数据与盘后数据需区分
    - 过期数据必须重新获取
  警告阈值: 超过15分钟

L2 日级 (有效期: 0-24小时):
  适用:
    - 每日收盘价
    - 日成交额
    - 当日新闻快讯
  处理策略:
    - 标注数据日期
    - 注意交易日与自然日区别
    - 周末/节假日数据需说明
  警告阈值: 超过1个交易日

L3 周级 (有效期: 0-7天):
  适用:
    - 周度资金流向
    - 周度行业动态
    - 短期市场分析
  处理策略:
    - 标注周期起止日期
    - 对比上周数据
  警告阈值: 超过7天

L4 月级 (有效期: 0-30天):
  适用:
    - 月度经济指标
    - 月度行业数据
    - 分析师研报
  处理策略:
    - 关注发布日期
    - 等待下月数据发布
  警告阈值: 超过30天

L5 季级 (有效期: 0-90天):
  适用:
    - 季度财务报表
    - 季度业绩分析
    - 行业季度报告
    - 机构评级（分析师评级）
  处理策略:
    - 标注报告期
    - 同比环比分析
    - 关注预披露/快报
    - **机构评级必须标注发布日期**
  警告阈值: 超过90天或超过下一报告期

L5 机构评级专项规范:
  定义: 券商/投行对股票的买入/持有/卖出评级及目标价
  有效期: 90天（建议），最长180天
  强制要求:
    - ✅ 必须标注评级发布日期（格式：YYYY-MM-DD）
    - ✅ 必须标注发布机构名称
    - ✅ 超过90天需加注时效警告
    - ✅ 超过180天视为过期，不建议引用
  展示格式:
    | 机构 | 评级 | 目标价 | 发布日期 | 时效状态 |
    |------|------|--------|----------|----------|
    | 摩根士丹利 | 增持 | 200美元 | 2025-09-29 | ✅ 有效 |
    | 美银证券 | 买入 | 114港元 | 2025-01-22 | ⚠️ 临近过期 |
  时效警告示例:
    - 0-30天: ✅ 最新（绿色）
    - 31-90天: ✅ 有效（绿色）
    - 91-180天: ⚠️ 较旧，建议更新（黄色）
    - >180天: ❌ 过期，仅供参考（红色）

L6 年级 (有效期: 0-365天):
  适用:
    - 年度财务报告
    - 年度战略规划
    - 长期行业研究
  处理策略:
    - 标注报告年度
    - 多年度趋势分析
  警告阈值: 超过1年

L7 永久级 (有效期: 不限):
  适用:
    - 公司注册信息
    - 政策法规原文
    - 历史事件记录
  处理策略:
    - 注意最新修订版本
    - 标注原始发布日期
  警告阈值: 无（但需确认是否有更新版本）
```

## 二、时间衰减权重模型

### 数学模型

```
时效性系数 = exp(-λ × 超期天数)

其中:
- λ: 衰减率常数
- 超期天数 = 当前日期 - 信息发布日期 - 有效区间
```

### 各等级衰减率

| 等级 | 有效区间 | 衰减率 λ | 半衰期 |
|------|----------|----------|--------|
| L1 | 15分钟 | 100 | ~10分钟 |
| L2 | 1天 | 10 | ~2.5小时 |
| L3 | 7天 | 3 | ~5.5小时 |
| L4 | 30天 | 1 | ~16小时 |
| L5 | 90天 | 0.3 | ~2.3天 |
| L6 | 365天 | 0.1 | ~7天 |
| L7 | 永久 | 0 | 无限 |

### 权重应用规则

```yaml
时效性系数解读:

  系数 ≥ 0.8:
    状态: ✅ 完全有效
    处理: 正常使用，高可信度

  0.5 ≤ 系数 < 0.8:
    状态: ⚠️ 部分有效
    处理: 可使用，但需标注可能存在偏差

  0.3 ≤ 系数 < 0.5:
    状态: ⚠️ 时效性存疑
    处理: 建议获取更新数据，或明确标注时间

  系数 < 0.3:
    状态: ❌ 可能过时
    处理: 必须更新，或仅作历史参考
```

## 三、迭代验证机制

### 验证流程

```yaml
Phase 0: 日期确认（前置强制步骤）
  步骤:
    1. 确认当前日期（YYYY-MM-DD）
    2. 确认当前年份（YYYY）
    3. 校验是否为财报披露期
    4. 设定报告基准时间戳
  输出:
    - 当前日期: 2026-03-14
    - 当前年份: 2026
    - 报告基准时间: 2026-03-14 15:30

Phase 1: 初始信息收集
  步骤:
    1. 构建带年份的搜索查询
    2. 获取信息内容
    3. 记录发布时间戳
    4. 判断信息类型
    5. 查询有效区间
    6. 计算时效性系数

Phase 2: 时效性检测
  步骤:
    1. 检查是否在有效区间内
    2. 计算超期天数
    3. 应用衰减模型
    4. 生成时效性状态

Phase 3: 过期处理
  条件: 时效性系数 < 0.8
  处理:
    - 系数 < 0.5: 触发更新查询
    - 系数 0.5-0.8: 标注警告
  更新策略:
    - 重新搜索最新信息
    - 对比新旧数据变化
    - 记录更新原因

Phase 4: 冲突处理
  条件: 新旧信息存在矛盾
  处理:
    1. 验证信息来源权威性
    2. 以官方/权威来源为准
    3. 标注变化原因
    4. 提示用户注意时间节点

Phase 5: 报告校验
  步骤:
    1. 汇总所有信息时效性
    2. 检查关键数据有效性
    3. 生成时效性声明表
    4. 提出数据更新建议
```

### 自动化检测脚本逻辑

```python
# 伪代码示例
def check_timeliness(info_type, publish_date, current_date):
    # 获取有效区间
    validity_period = VALIDITY_PERIODS[info_type]  # 天数

    # 计算超期天数
    days_elapsed = (current_date - publish_date).days
    overdue_days = max(0, days_elapsed - validity_period)

    # 计算时效性系数
    lambda_rate = DECAY_RATES[info_type]
    timeliness = math.exp(-lambda_rate * overdue_days)

    # 返回状态
    if timeliness >= 0.8:
        return "valid", timeliness
    elif timeliness >= 0.5:
        return "warning", timeliness
    elif timeliness >= 0.3:
        return "questionable", timeliness
    else:
        return "outdated", timeliness
```

## 四、特殊情况处理

### 1. 数据延迟发布

```yaml
场景: 财报延迟发布、数据源故障
处理:
  - 使用最近可获取的有效数据
  - 明确标注数据延迟情况
  - 说明预期发布时间（如已知）
示例:
  "2024年Q1财报尚未发布，使用2023年Q4数据。
   预计披露时间：2024年4月30日前"
```

### 2. 数据修正/重述

```yaml
场景: 财务数据重述、历史数据修正
处理:
  - 优先使用最新修正后的数据
  - 标注修正原因（如有）
  - 对比修正前后变化
示例:
  "该公司于2024年3月重述2023年财报，
   修正后净利润由5亿下调至4.2亿，
   原因：应收账款坏账准备增加"
```

### 3. 周期性数据

```yaml
场景: 季节性行业、周期性数据
处理:
  - 同比分析优先于环比
  - 标注行业周期特征
  - 使用周期调整后数据
示例:
  "该数据为Q4旺季数据，
   建议与去年同期Q4数据对比分析"
```

### 4. 不可比数据

```yaml
场景: 会计政策变更、业务重组
处理:
  - 调整后数据优先
  - 标注不可比原因
  - 提供历史追溯（如可获取）
示例:
  "2023年数据已追溯调整，
   原因：公司采用新收入准则"
```

## 五、报告中的时效性声明

### 标准格式

```markdown
## 数据时效性声明

**报告基准时间**: YYYY-MM-DD HH:MM

### 关键数据时效性

| 数据项 | 来源 | 发布时间 | 时效状态 | 建议 |
|--------|------|----------|----------|------|
| 股价 | 东方财富 | 2024-03-15 15:00 | ✅ 有效 | 无 |
| Q4财报 | 巨潮资讯 | 2024-03-10 | ✅ 有效 | 关注年报 |

| 行业数据 | 艾瑞咨询 | 2023-06-01 | ❌ 已过期 | 需更新 |

### 数据更新建议

1. 行业数据已过期超过6个月，建议获取最新行业报告
2. 分析师研报接近6个月有效期，可关注最新评级变动

### 免责声明

本报告基于公开信息整理，数据时效性以标注时间为准。
投资决策前请核实最新数据，过往表现不代表未来收益。
```

## 六、最佳实践

### 信息收集时

1. **记录时间戳** - 每条信息必须记录获取时间和来源发布时间
2. **验证时效性** - 收集时立即检查是否在有效区间内
3. **标注来源** - 附上原始链接便于验证

### 分析过程中

1. **优先最新** - 始终优先使用最新发布的数据
2. **交叉验证** - 多源验证数据一致性
3. **识别缺口** - 发现过期信息及时补充

### 报告输出前

1. **时效性扫描** - 检查所有关键数据的有效性
2. **生成声明** - 输出完整的时效性声明表
3. **更新建议** - 明确提示需要更新的数据项
