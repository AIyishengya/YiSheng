# 观察清单管理模块

## 概述

维护一个持续更新的标的观察清单，记录关注的标的、关键价位、触发条件和历史分析链接，方便快速回顾和跟踪。

## 触发方式

```
/financial-analysis 观察清单
/financial-analysis 添加观察 {标的名称}
/financial-analysis 移除观察 {标的名称}
/financial-analysis 更新观察清单
```

关键词：`观察清单`、`watchlist`、`关注列表`、`添加观察`、`移除观察`

## 清单文件位置

```
/财务分析/观察清单.md
```

## 自动维护规则

### 分析完成后自动更新

每次完成标的分析报告后，检查并更新观察清单：

```yaml
分析完成后:
  1. 读取 /财务分析/观察清单.md
  2. 若标的已在清单中:
     - 更新「当前价」「风险评分」「最近分析」链接
     - 更新「关注价位」（基于最新操作建议）
     - 更新「关键日历」中的事件
  3. 若标的不在清单中:
     - 提示用户是否加入观察清单
  4. 写回更新后的清单文件
```

### 关键日历维护

```yaml
事件管理:
  - 从分析报告中提取未来关键事件（财报日、政策会议等）
  - 已过期事件移入「历史事件」或删除
  - 按日期排序，最近的事件排在前面
```

### 回测评分集成

```yaml
回测完成后:
  - 将回测综合评分写入清单的额外字段
  - 在备注中标注预测准确度趋势
```

## 快速查看

当用户输入 `/financial-analysis 观察清单` 时：

1. 读取观察清单文件
2. 对活跃观察中的标的，快速搜索当前股价
3. 标注价格是否已触及关注价位
4. 高亮即将到来的关键事件（7天内）
5. 输出更新后的清单摘要
