# Skill 开发框架——参考模板

> SKILL.md 的配套文档——提供每种 skill 类型的可复制模板。

---

## 模板 1：参考型 Skill（最常见）

```markdown
---
name: <小写短横线名称>
description: 当<触发条件1>、<触发条件2>或<触发条件3>时使用此 skill。提供<提供的内容>。
language: zh-CN
origin: <ECC|community|custom>
---

# <标题>

<一句话概述此 skill 涵盖的内容。>

## 何时激活

- <具体场景 1>
- <具体场景 2>
- <具体场景 3>
- <具体场景 4>

## 核心原则

### 1. <原则名称>

<理由和解释。>

```<语言>
// ✅ 正确：<为什么好>
<代码示例>
```

```<语言>
// ❌ 错误：<为什么不好>
<代码示例>
```

### 2. <原则名称>

<理由和解释。>

```<语言>
// ✅ 正确示例
```

```<语言>
// ❌ 错误示例
```

## 代码示例

### <用例 1>

```<语言>
// 实用的、经过测试的示例
```

### <用例 2>

```<语言>
// 实用的、经过测试的示例
```

## 检查清单

- [ ] <可验证条目 1>
- [ ] <可验证条目 2>
- [ ] <可验证条目 3>
- [ ] <可验证条目 4>

## 常见陷阱

### 陷阱 1：<名称>
**症状：** <出了什么问题>
**原因：** <为什么会发生>
**修复：** <如何预防/解决>

### 陷阱 2：<名称>
**症状：** <出了什么问题>
**原因：** <为什么会发生>
**修复：** <如何预防/解决>

## 消费方 Agent

以下 agent 引用了此 skill：
- `<agent-name-1>` —— <用途>
- `<agent-name-2>` —— <用途>

## 参考资料

- 相关 skill：`<skill-1>`、`<skill-2>`
- <外部资源>
```

## 模板 2：组件型 Skill

```markdown
---
name: <小写短横线名称>
description: 当<触发条件>时使用此 skill。产出<制品类型>，用于<目的>。
type: component
language: zh-CN
---

## 目的

<此 skill 产出什么制品以及为什么重要。最多 2-3 句话。>

## 关键概念

### 框架

<框架/模板结构的描述。>

### 为什么有效

<支持此方法的证据或推理。>

### 反模式（这不是什么）

- ❌ <反模式 1>
- ❌ <反模式 2>

### 何时使用 / 何时不使用

**使用场景：**
- <条件 1>
- <条件 2>

**不适用场景：**
- <条件 1>
- <条件 2>

## 应用步骤

### 步骤 1：收集上下文

<首先需要收集什么信息。>

### 步骤 2：<节名称>

<如何填充此节。如适用，引用 template.md。>

### 步骤 3：<节名称>

<如何填充此节。>

### 步骤 N：审查与验证

<如何验证输出是否完整且正确。>

## 示例

### 示例 1：<描述性标题>

<完成的示例或指向 examples/ 目录的引用。>

## 常见陷阱

### 陷阱 1：<名称>
**症状：** / **后果：** / **修复：**

### 陷阱 2：<名称>
**症状：** / **后果：** / **修复：**

## 参考资料

### 相关 Skill
- [`<skill-1>`](../<skill-1>/SKILL.md) —— <关系>
- [`<skill-2>`](../<skill-2>/SKILL.md) —— <关系>

### 外部框架
- <来源或方法论参考>
```

## 模板 3：交互型 Skill

```markdown
---
name: <小写短横线名称>
description: 当<触发条件>时使用此 skill。通过自适应问题引导完成<决策/分析>。
type: interactive
language: zh-CN
---

## 目的

<此 skill 辅助什么决策或分析。2-3 句话。>

## 关键概念

### 框架

<决策模型或评分方法的描述。>

### 评分标准

<如适用，说明选项如何被评估。>

| 标准 | 权重 | 说明 |
|------|------|------|
| <名称> | <1-5> | <衡量什么> |

## 应用步骤

### 步骤 1：上下文问题

提出以下自适应问题（编号选项，支持多选）：

**Q1：<问题文本>**
1. <选项 A>
2. <选项 B>
3. <选项 C>
4. 其他（请说明）

**Q2：<问题文本>**
1. <选项 A>
2. <选项 B>
3. <选项 C>

### 步骤 2：评估

<如何根据答案进行评分或评估。>

### 步骤 3：交付建议

<建议输出的格式和内容。>

## 示例

### 示例交互

**背景：** <场景描述>
**答案：** Q1: <A>, Q2: <B>, Q3: <C>
**建议：** <预期输出>

## 常见陷阱

### 陷阱 1：<名称>
**症状：** / **后果：** / **修复：**

## 参考资料

### 相关 Skill
- [`<skill-1>`](../<skill-1>/SKILL.md) —— <关系>
```

## 验证清单（可打印）

```
SKILL 验证清单
==============
Skill 名称：____________________
类型：[  ] reference  [  ] component  [  ] interactive  [  ] workflow
日期：____________________

结构
[  ] name ≤ 64 字符，小写短横线
[  ] description ≤ 200 字符，中文，以"当……时使用此 skill"开头
[  ] language: zh-CN
[  ] type 与节结构匹配
[  ] 各节遵循类型特定模板
[  ] SKILL.md ≤ 5,000 token

可移植性
[  ] 无工具特定指令（Bash、Grep、Read、Write 等）
[  ] 无模型特定提示词（think harder、ultrathink 等）
[  ] 所有代码示例已标注编程语言
[  ] 可执行动作使用条件能力模式
[  ] 仅知识——无流程逻辑
[  ] 检查清单工具中立

可组合性
[  ] 交叉引用使用显式路径格式
[  ] 无循环依赖
[  ] 依赖已声明
[  ] 与 agent 无重复权威
[  ] 已添加"消费方 Agent"节

内容
[  ] "何时激活"有具体触发场景（非抽象描述）
[  ] 至少一个具体示例
[  ] 至少一个显式反模式
[  ] 无模糊或套话语言
[  ] 所有检查清单条目可验证

语言
[  ] 正文全部使用中文
[  ] 代码注释使用中文
[  ] description 使用中文
[  ] 技术术语首次出现有中文解释

兼容性（针对受限程度最高的 agent 测试）
[  ] 只读 agent 可使用（不需要写入/编辑/bash）
[  ] haiku 模型 agent 可解析结构
[  ] 不同工具生态可使用
[  ] 补充（而非替代）agent 流程

签核：____________________
```
